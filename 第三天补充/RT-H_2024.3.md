# RT-H: Action Hierarchies Using Language

## 基本信息

**标题**: RT-H: Action Hierarchies Using Language
**发表时间**: 2024年3月
**机构**: Google DeepMind, Stanford University
**作者**: Suneel Belkhale, Tianli Ding, Ted Xiao, Pierre Sermanet, Quan Vuong, Jonathan Tompson, Yevgen Chebotar, Debidatta Dwibedi, Dorsa Sadigh

## 核心贡献

### 创新点1：语言动作层次
- **问题描述**: 高级任务描述与低级动作之间差距大，难以共享数据。当任务变得语义上更加多样化时（例如"pick coke can"和"pour a cup"），在任务之间共享数据变得更加困难，因此学习从高级任务到动作的映射需要更多的演示数据。
- **解决方案**: 引入中间层"语言动作"（如"move arm forward"、"rotate arm right"、"close gripper"），在高级任务和低级动作之间建立层次结构。RT-H首先学习预测语言动作，然后基于这些语言动作和高级任务预测实际动作。
- **意义**: 在语言动作层面实现更好的数据共享和泛化。即使"pour a cup"和"pick up a coke can"在语义上不同，它们在语言动作层面完全重叠，直到物体被抓取。这使策略能够学习看似不同的任务之间低级动作的共享结构。

### 创新点2：人类干预学习
- **问题描述**: 机器人执行失败时难以纠正，传统的遥操作干预效率低且难以学习。
- **解决方案**: 允许人类提供语言动作纠正。当策略不是100%成功时，人类可以提供自己的语言动作纠正。更重要的是，机器人可以从这些人类纠正中学习，完全在语言动作领域进行。
- **意义**: 机器人可以从语言干预中学习，比遥操作干预更有效。这使机器人能够学习新的语言动作，并利用这些新的语言动作预测更好的动作。一旦人类完成干预，RT-H恢复像以前一样预测语言动作。

### 创新点3：自动化语言动作提取
- **问题描述**: 手动标注语言动作成本高，需要大量人工工作。
- **解决方案**: 从机器人本体感觉自动提取2500+语言动作。RT-H不需要人工标注即可获得丰富的语言动作库。
- **意义**: 无需人工标注即可获得丰富的语言动作库，大大降低了标注成本。语言动作不是固定的原语，而是在当前任务和场景的上下文中学习，具有上下文性和灵活性。

## 技术要点

### 架构设计
```
输入层:
  - 机器人摄像头图像
  - 高级任务描述（如"close pistachio jar"）

第一级预测（语言动作）:
  - Vision Language Model (VLM)
  - 预测语言动作（如"move arm forward"、"rotate arm right"）
  - 基于视觉上下文和任务描述

第二级预测（实际动作）:
  - 基于语言动作和高级任务
  - 预测机器人动作（7维向量）
  - 条件化于语言动作

人类干预:
  - 语言动作纠正
  - 机器人学习新语言动作
  - 恢复正常预测
```

### 语言动作示例
- "move arm forward" - 移动手臂向前
- "rotate arm right" - 旋转手臂向右
- "close gripper" - 关闭夹爪
- "grasp can" - 抓取罐子
- "move arm up" - 移动手臂向上

### 方法论

#### 动作层次结构
1. **高级任务**: 自然语言描述的任务（如"close pistachio jar"）
2. **语言动作**: 中间层，描述低级运动的细粒度短语
3. **实际动作**: 机器人的低级动作（7维向量）

#### 训练策略
- **两阶段预测**: 先预测语言动作，再预测实际动作
- **条件化**: 实际动作预测条件化于语言动作
- **上下文学习**: 语言动作在任务和场景上下文中学习
- **人类干预**: 支持语言动作纠正和学习

#### 语言动作特性
- **上下文性**: "move arm forward"单独不传达移动速度或确切方向向量；这取决于任务和观察
- **灵活性**: 语言动作可以根据上下文调整
- **可组合性**: 语言动作可以组合形成复杂行为
- **可学习性**: 机器人可以从人类干预中学习新的语言动作

## 实验结果

### 性能表现
- **vs RT-2**: 提升15%
- **vs IWR**: 提升50%
- **干预学习**: 语言动作纠正接近完美成功率

### 任务示例
1. **关闭开心果罐**: "close pistachio jar"
2. **抓取可乐罐**: "pick coke can"
3. **倒杯子**: "pour a cup"
4. **移动手臂**: "move arm forward"、"rotate arm right"

### 数据共享效果
- **语义不同任务**: "pour a cup"和"pick up a coke can"在语义上不同
- **语言动作重叠**: 在语言动作层面完全重叠，直到物体被抓取
- **泛化能力**: 在语言动作层面实现更好的数据共享和泛化

### 人类干预效果
- **纠正能力**: 人类可以提供语言动作纠正
- **学习能力**: 机器人可以从人类纠正中学习
- **性能提升**: 比遥操作干预更有效

## 优缺点分析

### 优点
1. **层次结构**: 语言动作作为中间层，提升数据共享和泛化
2. **人类干预**: 支持语言动作纠正，易于人类使用
3. **自动提取**: 无需人工标注即可获得语言动作库
4. **上下文学习**: 语言动作在任务和场景上下文中学习
5. **可组合性**: 语言动作可以组合形成复杂行为
6. **性能提升**: 比RT-2提升15%，比IWR提升50%

### 缺点
1. **两阶段**: 需要先预测语言动作，再预测实际动作，增加了计算复杂度
2. **依赖VLM**: 基于RT-2，继承其限制（如计算需求、离散化损失）
3. **语言动作提取**: 自动提取的语言动作可能不如人工标注准确
4. **上下文依赖**: 语言动作的性能依赖于任务和场景上下文

## 学习笔记

### 关键洞察
1. **语言的作用**: 语言不仅能表达高级任务，还能表达如何执行任务——更细粒度的表示，更接近低级动作
2. **层次结构的价值**: 在高级任务和低级动作之间建立中间层，可以更好地捕获不同任务之间的相似性
3. **人类干预的新范式**: 语言动作纠正比遥操作干预更有效，机器人可以从语言干预中学习

### 技术要点
1. **语言动作**: 描述低级运动的细粒度短语，如"move arm forward"、"rotate arm right"
2. **上下文性**: 语言动作不是固定的原语，而是在当前任务和场景的上下文中学习
3. **两阶段预测**: 先预测语言动作，再预测实际动作，条件化于语言动作

### 实验发现
1. **数据共享**: 在语言动作层面实现更好的数据共享，即使语义不同的任务在语言动作层面重叠
2. **性能提升**: 比RT-2提升15%，比IWR提升50%
3. **人类干预**: 语言动作纠正接近完美成功率，机器人可以从人类纠正中学习

### 未来方向
1. **更多语言动作**: 扩展语言动作库，覆盖更多任务和场景
2. **多模态干预**: 支持更多类型的人类干预（如手势、语音）
3. **在线学习**: 实现在线学习，实时适应新的语言动作
4. **跨机器人**: 将语言动作层次扩展到更多机器人平台

## 总结

RT-H通过引入语言动作层次，在高级任务和低级动作之间建立了中间层，实现了更好的数据共享和泛化。语言动作不仅提供了更好的数据共享，还支持人类通过语言动作纠正机器人，机器人可以从这些纠正中学习。实验表明，RT-H比RT-2提升15%，比IWR提升50%，展示了语言动作层次的价值。

RT-H的核心创新在于：
1. **语言动作层次**: 在高级任务和低级动作之间建立中间层
2. **人类干预学习**: 支持语言动作纠正，机器人可以从人类纠正中学习
3. **自动提取**: 无需人工标注即可获得丰富的语言动作库

RT-H为机器人学习提供了新的范式，通过语言动作层次和人类干预学习，实现了更灵活、更强大的机器人策略。
